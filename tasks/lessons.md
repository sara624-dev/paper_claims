# Lessons — 論脈 RONMYAKU

## 2026-07-05 MVP 構築時

### PDF 読解は pdftotext を使う
- **事象**: この環境（Pi コンテナ）では Claude Code の Read ツールによる PDF レンダリングが使えない
- **対応**: `pdftotext`（poppler-utils）でテキスト化して読む。SKILL.md 手順3に反映済み
- **ルール**: PDF 処理を含むスキルを書くときは、Read の PDF 対応を前提にせず pdftotext を第一選択にする

### 関係（relation）は「実在する反証」だけを張る
- **事象**: E2E 検証で CoT / Self-Consistency / Cannot-Self-Correct の3本を取り込んだが、この組み合わせには真の contradicts が存在しなかった（Huang 2310.01798 の反証対象は Reflexion / Self-Refine 等の未収載論文）
- **対応**: 無理に赤エッジを作らず、反証対象の Self-Refine（2303.17651）を追加取り込みして実在の係争関係で検証した
- **ルール**: 「グラフに全種類のエッジを出したい」という検証都合でデータを歪めない。contradicts が無いのはそれ自体が正しい研究状況の反映。反証関係を見たければ反証している論文を取り込む

### UI は必ずスクリーンショットで見た目を検証してから完了とする（ユーザー指摘 2026-07-05）
- **事象**: グラフビューを HTTP 200 / API 出力の確認だけで「完成」と報告し、ユーザーから「ビジュアライズ部分が非常に残念」と指摘を受けた。cose レイアウトに素のノードを流しただけで、視覚品質を一度も見ていなかった
- **ルール**: UI を含む成果物は Playwright 等のヘッドレスブラウザで**モバイル・デスクトップ両ビューポートのスクリーンショットを撮り、自分の目で確認してから**完了と報告する。「レンダリングされる」と「見やすい」は別物
- **ルール**: グラフ可視化は汎用物理レイアウト任せにせず、ドメイン構造（論文→クレーム、時系列）をレイアウトに反映する

### コンテナ内パスとホストパスの混同に注意
- **事象**: セッション内の `!` コマンドはコンテナ内で実行されるため、ホストパス前提の sudo 手順が `No such file or directory` になった
- **ルール**: systemd 適用などホスト側作業の手順は「Pi に SSH して実行」と明記する。コンテナ内は `/workspace/...`、ホストは `/home/YOUR_USER/workspace/...`

### 判断材料の提示・比較は文脈込みで行う（ユーザー指摘 2026-07-06）
- **事象**: 課題の同一性確認で quote 断片だけを並べた AskUserQuestion を出し、「文脈が落ちて判断できない・ミスリードを生む」と指摘を受けた
- **ルール**: 主張・課題の抽出時に `context_ja`（指示語の解決・前提条件・前後の議論）を必ず保存する。関係judgment・同一性判定はインデックスの一行サマリで行わず、必ず quote + context + evidence を読んでから。ユーザーに紐付け確認を求めるときも文脈を並べて提示してから聞く
- **一般化**: 抽出時（原文全体が見えている唯一の機会）に文脈を固定しないと、後段のすべての判断が断片ベースになり劣化する

### arXiv API はまれに別論文を返すことがある
- **事象**: `export.arxiv.org/api/query?id_list=` が一度、別 ID の論文メタデータを返した（サブエージェント報告）
- **ルール**: メタデータ取得後、PDF 1ページ目のタイトルと一致することを必ず確認する
